Support

Shorty support

Run these checks first. Email shorty@peyton.app with the macOS version, app name, and shortcut if Shorty still does not behave as expected.

Download and checksum

Download Shorty from the latest release. Each app archive and source archive is paired with a .sha256 file.

shasum -a 256
shorty-<version>-macos.zip

The command output should match the checksum line published with the release asset.

Accessibility setup

Grant access

Open Shorty, choose Open Accessibility Settings, then allow Shorty in System Settings, Privacy & Security, Accessibility.

Check again

Return to the Shorty menu and choose Check Again. Shorty retries the event tap without requiring a quit and reopen.

What Shorty needs

Accessibility access lets Shorty read menu item names, inspect published shortcuts, and listen for the key combinations it remaps.

Optional browser bridge

Native app shortcut remapping works without the browser bridge. Install the bridge only when you want supported web-app adapters in Chrome, Chrome Canary, Chromium, Brave, Edge, or Vivaldi.

just install-browser-bridge
EXTENSION_ID=<extension-id>
BROWSERS=chrome,brave,edge

The bridge writes native messaging manifests under each selected browser's Application Support directory and places the helper at .build/browser-bridge/shorty-bridge.

just uninstall-browser-bridge
BROWSERS=chrome,brave,edge

Troubleshooting

Shorty is not running

Open Shorty from Applications and look for the menu bar icon. If it is missing, launch the app again before testing shortcuts.

Unsupported app pass-through

If the front app has no adapter, Shorty leaves the key press alone and shows that the app is unsupported in the popover.

Bridge not connected

Confirm the browser extension is loaded, the native messaging manifest uses the same extension ID, and Shorty is already running.

Source

The project source is available at github.com/peyton/shorty. Public direct-download releases include a shorty-<version>-source.tar.gz archive for the matching version.
