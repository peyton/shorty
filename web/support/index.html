Email shorty@peyton.app with the macOS version, app name, and the command or shortcut that did not match.

Setup checks

Accessibility access

Shorty needs macOS accessibility access so it can read menu items in the active app. Open System Settings, Privacy & Security, Accessibility, then allow Shorty.

Menu permissions

If an app does not expose menu item shortcuts through macOS, Shorty may not be able to list every command.

Browser bridge

Browser integrations rely on the native messaging bridge included with the app source.

Source

The project source is available at github.com/peyton/shorty. Build and validation commands are documented in the repository.
